默默忙碌了大半年

今天看某童鞋写了个心情感慨临近30而立，悚然发现我也接近了。而且前段时间华为裁大龄员工事件闹得沸沸扬扬了，虽然不知真假，但是依然被惊了下，早期虽然想过该问题却刻意忽略了。趁着周末思考下，技术人员到底该何去何从。

first

从2009年工作至今，已经过去了七八个年头，期间因为互联网的兴起也算是顺风顺水，没有碰到太大的压力。公司几经变更，负责的事项也不断地改变，但是总体而言还是在技术范围内波动。早期的想法便是一直在这个领域深入下去，不断地跟上时代的发展。

以前看着技术人员年龄不能超过30，到不能超过35，到如今的40，一直当做是笑话在看待，直到华为的裁员事件，让我们这批乐观派不得不自我检讨了下。

second

纵观这十年的发展，技术人员的要求在不断地降低。在十年前我认为技术人员应该是越往底层走越厉害，那才是技术的终极方向。接着我认为技术人员应该广度上拥有足够的视野，否则容易被淘汰。但是事实却是底层的东西几经变更，大部分的童鞋其实都不再去思考了。而广度方面，随着流水线式的工作方向的兴起，或者说行业方向的精细化，越来越多的童鞋只会了解自己的那一片小领域。

很难说这是好事还是坏事，但是根据历史的定律，行业越来越成熟后，人员要求其实反而是在逐步降低的，就如同云计算出现后，公司的运维人员需求变少了。各种开源项目及平台的出现也让项目的复杂度逐步降低，人员可能只需要以前的几分之一却能做出之前几倍的事情。

而在学习力方面，多年工作经验的人其实学习起来比新入行的童鞋是要快，但是扛不住人家精力旺盛，并且新技术出现的速度在逐步地加快，因此反而是有经验的童鞋处劣势。另外一方面，有经验的童鞋薪酬一般较高，公司的负担较重，而基础性的活说实话3年与7年经验可能差异不是很明显，而复杂性的活根据金字塔定律，需求的人员比起基础的活少得多，因此两级分化下，员工只能去选择转向管理或者专家或者架构师。但是有多少公司需要那么多管理或专家或架构呢，有多少童鞋深度和广度又能赶上呢？

企业说白了依然是赤裸裸的市场环境，只有撑得下去才有员工，因而我们创造的价值必须是大于我们的收入的，否则便有风险。但是大部分的企业想跟上大家的薪资成长速度还是比较难的，当然有一些新方向或者较窄的行业比较特殊。就如同前两年的P2P这个业务方向的井喷时期，为很多的童鞋带来了一批福利，不过倒了后却有一批人因为找不到匹配薪资的问题而找不到工作。而较窄的行业招收的人偏少，并且要求还是颇高的。

third

感觉这些文字有点沉重，心情都压抑了。

难道说真的到了40后我们就都要退休了？往好的方面想，计算机行业不可能会突然就完全不见掉。

因此金字塔上方的一波人其实不会受到太大影响，可能也就是收入的增长速度会变慢而已。

而处于中间的一波人，有可能会受到比较大的影响，但是市场的优胜略汰决定了部分基础扎实学习力强的童鞋也不会受到太大的影响，而剩下的童鞋可能会因为不匹配薪资和年龄原因被抛弃。

金字塔底部的童鞋也就是刚刚入行的童鞋们，可能只是实习机会等变少了，因为企业不再花太多时间去培训了，而进入中层的时间却会被逐步拉长，因为中层积压现象严重，因此薪资成长福利会越来越差。但是机会也很多，因为新技术等层出不穷，学习力旺盛的新人们还是有着大把机会的，只是需要些运气元素。

four

如何避免被淘汰呢？

先说下题外话。关于面试，其实也已经面了五年了，期间见过各个不同层次的童鞋。感觉上，其实很多童鞋可能都是被公司给害了…..为什么这么说呢？有些公司没有什么技术氛围，其实做了3年增删改查改业务逻辑，你除了码代码速度快了，业务逻辑熟悉了，剩下的没有自学就相当于浪费了2年多。而增长快的公司，你会面临各种坑，各种需要你不断去学习才能处理的新问题，你的一年成长可能比两三年的自学还有用处，虽然你也会被逼得在业余时间不断自学。

so？吐槽完毕，总结下还是在这几个方面：

  • 大方向选择
  • 自学
  • 自我定位选择
  • 创业
  • 换行业

所谓的大方向选择便是把精力逐步往一些逐步热门的方向上靠，虽然有一定的选择失误风险。逐步热门的例如像大数据方向、机器学习方向、人工智能方向、VR方向等，虽然招聘量看起来比较少，入门的门槛也高了一些，但是薪资还是令人满意的，供需关系依然是供不应求。而几乎每年都有一些新的方向在出现，如何选择就看个人了。

自学这块可能只要是个热爱计算机的程序猿都是爱自学的，但是方法一定要选好，同时加大实践量以及逐步从学到分享上去靠，只有这样才是真正地学到了新东西。如果碰巧类似我们公司这样子经常有新业务需求和一些奇怪的事情，你会发现现学现卖的情况下，你的技能增长水平会远远比只看书或者只看分享的快得多，一年经验可以当两年使用。而在新技术层出不穷的今日，也只有自学才能让你跟上时代的节奏。

所谓的自我定位，便是你希望自己往哪个岗位发展，如果单纯拼编码效率，也许你是新人的两倍，但是你的薪资却可能是人家的六七倍。这样子公司的选择就不言而喻了，但是职场有句话，就是让自己成为不可或缺的人(实际上很多时候做的事情是反过来的，让机器能尽量替代自己ㄒoㄒ)，因此你应该在团队里找到定位，如果希望一直钻研技术，那就往核心工程师(核心业务逻辑精通)、某项专家(某项或者某几项技能精通)、架构师(类似我这样子打杂)方向发展，或者也可以选择往管理岗方向(业务逻辑精通，管理能力五颗星)发展。

创业就是个复杂的话题了，毕竟我还是木有任何信心去做这个事情的。但是不可否认是个出路。

而对于学习比较没兴趣的童鞋或者觉得自己不太喜欢码代码的话，其实还是建议直接转行吧，跳到公务员事业单位啥的，至少未来有个保障- -

five

其实到最后，我觉得我也难以说10年后我是否也会面临失业的危机，毕竟时代的发展与变迁总是超出当前年代的人们的想法。

但是纵观非本行业的童鞋而言，我们却也有很多新的机遇，毕竟这是个崭新的年代，并且我们已经算是在这波浪潮中的部分既得利益者了。

既然没法子保证自己不会失业，只能努力地尽量让自己踮着脚尖凑到金字塔的顶端，让自己不成为被淘汰的那一拨人，然后开心点度过每一天吧~ 毕竟工作只是生活的一小部分，不要因为未来而让当前的生活变质~

last

吐槽了不少，希望过几年来看这段文字的时候，会发现我是杞人忧天了。大家如果对未来有啥新的看法，也欢迎来信交流 zs1379@qq.com
